# SVG Illustration Generation via LLM

**Date:** 2026-05-01  
**Status:** Approved

## Goal

Replace raster PNG image generation (HuggingFace image model) with SVG code generated by an LLM. The SVG is stored as a `.svg` file in the uploads directory; the frontend renders it as before via `<img src="...">`.

## Architecture

Two-phase reactive pipeline inside `IllustrationService`:

```
Story
  │
  ▼ Phase 1 — scene extraction
  PromptBuilder.buildSceneExtractionUser(story.content)
  + scene-extraction-system.txt
  → HuggingFaceClient.generateText()   [HF_SVG_MODEL]
  → sceneDescription: String
  │
  ▼ Phase 2 — SVG generation
  PromptBuilder.buildSvgUser(story, sceneDescription)
  + svg-system.txt
  → HuggingFaceClient.generateText()   [HF_SVG_MODEL, new]
  → rawResponse: String
  │
  ▼ Phase 3 — extract + store
  extractSvgTag(rawResponse) → svgText: String
  ImageStorageService.saveSvg(story.id, svgText) → "/uploads/{id}.svg"
```

## Prompt Templates

### `prompts/scene-extraction-system.txt`
Short system prompt instructing the model to output exactly one sentence describing the main visual scene (setting, character, action). No preamble, no extra punctuation.

### `prompts/svg-system.txt`
The full SVG expert system prompt specifying:
- Flat vector children's storybook style
- viewBox 800×600, max 55 elements, no filters/images/text/masks
- Bright daytime background, lollipop trees, cloud shapes, geometric characters
- Character color: model infers an appropriate warm hex from the character name/type (no `{characterColor}` placeholder — instruction is embedded in the system prompt)

### `PromptBuilder.buildSvgUser(story, sceneDescription)`
Fills user prompt with:
- `{sceneDescription}` — from Phase 1 LLM output
- `{character}` — `story.characters[0]`
- Supporting characters — `story.characters[1..n]` joined, added as a line in the prompt
- `{ageGroup}` — `story.ageGroup`
- `{firstTwoSentencesOfStory}` — first two `.`-split segments of `story.content`

Character color is not a placeholder; the system prompt instructs the model to choose an appropriate warm hex based on the character name.

## Reactive Chain

```java
public Mono<String> generateIllustration(Story story) {
    String fallback = story.getCharacters().get(0)
                    + " in a magical scene from " + story.getTitle();

    return hfClient.generateText(promptBuilder.buildSceneExtractionSystem(),
                                 promptBuilder.buildSceneExtractionUser(story.getContent()))
        .onErrorReturn(fallback)
        .map(scene -> scene.isBlank() ? fallback : scene)
        .flatMap(scene ->
            hfClient.generateText(promptBuilder.buildSvgSystem(),
                                  promptBuilder.buildSvgUser(story, scene)))
        .map(this::extractSvgTag)
        .flatMap(svg -> imageStorage.saveSvg(story.getId(), svg));
}
```

`extractSvgTag` finds the first `<svg` and last `</svg>` in the raw response. Returns `PLACEHOLDER_SVG` (a minimal valid inline SVG) if neither is found.

## Error Handling

| Failure point | Behaviour |
|---------------|-----------|
| Scene extraction API error | Fall back to `"{characters[0]} in a magical scene from {title}"` |
| Scene extraction returns blank | Same fallback |
| SVG generation API error | Propagate — caller already handles `IllustrationService` errors |
| Response contains no `<svg>` tag | Use `PLACEHOLDER_SVG` constant |

## Configuration

```
HF_SVG_MODEL=<model-id>   # e.g. mistralai/Mistral-7B-Instruct-v0.3
```

Added to `.env.example`. `HfProperties` reads it as `svgModel`.
