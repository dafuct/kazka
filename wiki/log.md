# Wiki Log

Append-only chronological record of operations on the wiki. Each entry begins with `## [YYYY-MM-DD] <op> | <description>` so it's parseable with `grep "^## \[" log.md | tail -N`.

Operations:
- `ingest` — a source was processed into the wiki.
- `query` — a question was answered against the wiki (typically only logged when the answer was filed back as synthesis).
- `lint` — a health check was run.
- `schema` — the schema was modified.
- `shard` — an index was sharded.

## [2026-05-27] lesson | candidate: user_entitlements.product_id FK blocks non-catalog grants — Spec I gift redemption tried to insert a row with productId="gift" which violated fk_user_entitlements_product (no matching subscription_products row). Migration 019 made product_id nullable + dropped the FK. Lesson: when a column is FK-bound to a catalog table, future entitlement sources that don't map to catalog products (gifts, comps, promo, manual admin) need either (a) a sentinel catalog row, (b) nullable + drop FK (chosen), or (c) a separate entitlement_grants table. The "drop FK" choice is acceptable here because state+source already disambiguate, but future sources should be reviewed before adding more.
